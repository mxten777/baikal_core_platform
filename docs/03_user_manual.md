# BAIKAL Core Platform — 사용자 매뉴얼 (고객사 운영자)

> 대상: 고객사 담당자 (site_admin / editor)
> 버전: 1.0.0 | 작성일: 2026-03-13

---

## 1. 이 매뉴얼의 대상 독자

이 매뉴얼은 **귀사의 웹사이트를 직접 운영하시는 담당자**를 위한 안내서입니다.

- 기술적 지식이 없어도 따라할 수 있도록 작성되었습니다.
- 개발자 도움 없이 콘텐츠를 등록·수정·발행하고 미디어를 관리할 수 있습니다.

---

## 2. 접속 방법

### 관리자 페이지 주소
```
https://귀사도메인.com/admin
```

담당자에게 전달받은 **이메일 계정**으로 로그인합니다.

### 로그인 순서
1. 브라우저에서 위 주소로 이동
2. 이메일과 비밀번호 입력
3. **로그인** 버튼 클릭
4. 대시보드 화면으로 이동되면 정상입니다.

> **비밀번호를 잊으셨나요?**
> 로그인 화면의 **비밀번호 찾기** 링크를 클릭하거나, 플랫폼 담당자(Baikal Systems)에게 연락하세요.

---

## 3. 대시보드

로그인 후 처음 표시되는 화면입니다.

| 영역 | 내용 |
|------|------|
| 현재 사이트 정보 | 사이트명, 도메인, 활성화된 기능 확인 |
| 빠른 작업 | 자주 쓰는 메뉴로의 바로가기 버튼 |

## 4. 콘텐츠 관리

### 4-1. 콘텐츠 목록 보기

왼쪽 메뉴에서 **[콘텐츠]** 클릭

- 사이트의 모든 콘텐츠가 목록으로 표시됩니다.
- 상태 표시:
  - **발행됨(published)**: 현재 웹사이트에서 공개 중
  - **초안(draft)**: 저장되어 있지만 아직 공개 안 됨
  - **보관됨(archived)**: 공개 중지 상태

---

### 4-2. 새 글(포스트) 작성하기

1. 상단의 **[새 콘텐츠]** 버튼 클릭
2. 타입을 **포스트**로 선택
3. 제목 입력 → 주소(slug)가 자동으로 만들어집니다.

    > 예: 제목 "2026년 신제품 안내" → 주소 `2026-sinjepu-angae`

4. 본문 내용 작성
5. 저장 방법 선택:
   - **[초안 저장]**: 지금 당장 공개하지 않고 임시 저장
   - **[즉시 발행]**: 작성 즉시 웹사이트에 공개

---

### 4-3. 페이지 만들기

회사 소개, 서비스, 오시는 길 등 **고정 페이지** 제작에 사용합니다.

1. **[새 콘텐츠]** 클릭 → 타입을 **페이지**로 선택
2. 제목과 본문 입력 후 **[즉시 발행]** 클릭
3. 완성된 페이지에 접속: `https://귀사도메인.com/<주소>`

---

### 4-4. 프로젝트 등록 (포트폴리오)

포트폴리오 모듈이 활성화된 경우 사용 가능합니다.

1. **[새 콘텐츠]** 클릭 → 타입을 **프로젝트**로 선택
2. 프로젝트 제목 · 설명 · 내용 입력
3. 발행 후 `/portfolio` 페이지에 자동 등록됩니다.

---

### 4-5. 글 수정하기

1. 콘텐츠 목록에서 수정할 항목의 **[편집]** 클릭
2. 내용 수정 후 저장
3. 이미 발행된 글을 수정하면 변경 내용이 즉시 반영됩니다.

---

### 4-6. 주소(slug) 안내

주소(slug)는 웹사이트에서 해당 글에 접근하는 경로입니다.

```
https://귀사도메인.com/blog/주소
https://귀사도메인.com/portfolio/주소
https://귀사도메인.com/주소  (페이지의 경우)
```

> **주의**: 이미 발행된 글의 주소를 변경하면, 기존 주소로 방문하는 사람에게 404 오류가 발생합니다. 주소는 처음 작성 시 신중히 설정하세요.

---

## 5. 미디어 관리

### 5-1. 이미지 업로드

1. 왼쪽 메뉴 **[미디어]** 클릭
2. **[파일 업로드]** 버튼 클릭
3. 업로드할 이미지/파일 선택
4. (선택사항) alt 텍스트 입력:
   - 예: "2026년 신제품 라인업 전시 사진"
   - 검색엔진 노출 및 시각장애인 접근성에 도움이 됩니다.
5. 업로드 완료 후 미디어 라이브러리에 즉시 표시됩니다.

### 5-2. 이미지 크기 권장사항

| 용도 | 권장 크기 |
|------|----------|
| 썸네일 / 카드 이미지 | 800 × 450 px |
| 풀 배너 / 히어로 이미지 | 1920 × 800 px |
| OG 이미지 (SNS 공유) | 1200 × 630 px |
| 파비콘 | 32 × 32 px |

### 5-3. 지원 파일 형식

- **이미지**: JPG, PNG, WebP, GIF
- **문서**: PDF
- **동영상**: MP4, WebM *(용량 제한: Supabase 플랜에 따라 다름)*

## 6. 문의 폼 확인

고객이 웹사이트에서 문의를 남긴 경우 확인합니다.

1. 왼쪽 메뉴 **[폼 제출]** 클릭
2. 접수된 문의 목록 확인 (날짜, 이름, 이메일, 내용)
3. 중요한 문의는 별도로 기록하거나 담당자에게 전달하세요.

> **이메일 알림**: 현재 버전에서는 실시간 이메일 알림 기능은 지원하지 않습니다. 정기적으로 직접 확인해 주세요.

## 7. 블로그 운영 가이드

블로그 모듈이 활성화된 경우 포스트를 통해 뉴스·공지·정보를 발행합니다.

### 효과적인 블로그 운영 팁

| 항목 | 권장 사항 |
|------|----------|
| 발행 주기 | 주 1회 이상 정기 발행 권장 |
| 제목 | 검색에 잘 노출되도록 키워드 포함 (예: "2026년 서비스 요금 안내") |
| 본문 길이 | 최소 300자 이상 (SEO 효과) |
| 이미지 | 본문 상단에 대표 이미지 1장 이상 권장 |
| 주소(slug) | 짧고 명확하게 (예: `2026-service-price`) |

## 8. SEO(검색엔진 최적화) 기본 설정

> SEO 설정은 플랫폼 담당자(Baikal Systems)를 통해 변경을 요청하세요.

관리자가 설정할 수 있는 SEO 항목:

| 항목 | 위치 | 설명 |
|------|------|------|
| 사이트 제목 | site_settings.meta.title | 브라우저 탭·검색결과에 표시 |
| 사이트 설명 | site_settings.meta.description | 검색결과 미리보기 텍스트 |
| OG 이미지 | site_settings.meta.ogImage | SNS 공유 시 표시되는 이미지 |
| 개별 콘텐츠 | ContentEditorPage | 콘텐츠별 제목이 자동 적용됨 |

## 9. 자주 묻는 질문 (FAQ)

**Q. 발행한 글이 웹사이트에서 안 보여요.**
> 상태가 published인지 확인하세요. 콘텐츠 목록에서 초록색 배지로 표시됩니다.

**Q. 이미지를 업로드했는데 본문에 삽입이 안 돼요.**
> 현재 버전의 에디터는 텍스트 기반입니다. 이미지 본문 삽입 기능은 다음 버전 예정입니다.
> 현재는 이미지 URL을 복사하여 직접 마크다운 형식으로 입력 가능합니다: `![설명](URL)`

**Q. 실수로 글을 삭제했어요.**
> 콘텐츠는 즉시 삭제되지 않고 archived(보관) 상태로 변경됩니다.
> 플랫폼 담당자에게 복원 요청을 하세요.

**Q. 관리자 페이지가 로딩이 안 돼요.**
> 브라우저 캐시를 삭제(Ctrl+Shift+Delete)하고 재시도하세요.
> 문제가 지속되면 플랫폼 담당자(Baikal Systems)에게 연락하세요.

**Q. 모바일에서도 관리자 페이지 사용이 가능한가요?**
> 기본적인 내용 조회는 가능하지만, 콘텐츠 작성은 PC 환경을 권장합니다.
