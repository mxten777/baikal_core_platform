# BAIKAL Core Platform — 사업계획서 / 제안서

> 버전: 1.0.0 | 작성일: 2026-03-13
> 대상: 잠재 고객사 · 투자자 · 파트너

---

## 1. 요약 (Executive Summary)

**BAIKAL Core Platform**은 중소기업·전문가·기관이 **빠르고 안정적으로 웹사이트를 운영**할 수 있도록 설계된 **멀티사이트 웹 플랫폼 엔진(SaaS)**입니다.

| 항목 | 내용 |
|------|------|
| 제품 유형 | SaaS 형 웹 플랫폼 (멀티테넌트) |
| 핵심 가치 | 개발비 절감 · 빠른 구축 · 통합 운영 |
| 목표 고객 | 중소기업, 전문가, 의료기관, 쇼핑몰 운영자 |
| 기술 기반 | React 19 + Supabase + Vercel |
| 상용화 상태 | MVP 개발 완료, 2개 사이트 운영 중 |

---

## 2. 문제 정의 (Problem Statement)

### 현재 웹사이트 구축 시장의 문제점

| 문제 | 기존 방식 결과 |
|------|--------------|
| 구축 비용 과다 | 단순 기업 사이트에 수백만 원 소요 |
| 긴 개발 기간 | 평균 4~8주 소요 |
| 운영 비용 지속 발생 | 업데이트마다 개발사 의존 |
| 보안·장애 대응 취약 | 소규모 개발사는 24/7 대응 불가 |
| 각 사이트 별 분리 관리 | 다수 사이트 운영 시 비효율 폭증 |

---

## 3. 솔루션 (Solution)

### BAIKAL Platform의 접근법

```
기존 방식: 고객사마다 별도 개발 → 높은 비용, 낮은 품질, 느린 속도
           ↓
BAIKAL 방식: 하나의 엔진 위에 데이터 설정만으로 사이트 생성
            → 낮은 비용, 높은 품질, 빠른 속도 (1~3일 개설)
```

### 핵심 차별화 요소

1. **멀티테넌트 아키텍처**: 수십~수백 개의 사이트를 단일 코드베이스로 운영
2. **모듈 ON/OFF**: 블로그, 포트폴리오, 문의폼 등을 고객사별로 조합
3. **템플릿 시스템**: 업종별 최적화 디자인을 즉시 적용
4. **자체 CMS**: 고객사 담당자가 직접 콘텐츠를 관리 (개발사 의존 불필요)
5. **보안 설계**: RLS + RBAC으로 데이터 격리 및 권한 관리

## 8. 도입 절차 (고객사 관점)

```
1단계 (1일)  — 요구사항 파악
  └─ 업종, 필요 기능, 도메인, 디자인 컨셉 협의

2단계 (1일)  — 설정 및 구성
  └─ DB에 사이트 등록, 템플릿·모듈 선택, 기본 설정 입력

3단계 (1~2일) — 초도 콘텐츠 입력
  └─ 회사 소개, 서비스, 팀 소개 등 기본 페이지 구성

4단계 (0.5일) — 검수 및 오픈
  └─ 고객사 최종 확인 → 도메인 연결 → 공개

5단계 (상시) — 운영 지원
  └─ 고객사 자체 콘텐츠 운영 + 기술 지원
```
